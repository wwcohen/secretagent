1. **What SPECIFIC failure pattern did you identify?**
   The main entry point `calculate_medical_value` is natively an `@interface` returning a `float`. Without a `@implement_via('direct')` block, the orchestrator defaults to a zero-shot tool schema strictly expecting a float. This forces the LLM to guess the final number without Step-by-Step reasoning (Chain of Thought), leading to predictable math/threshold errors on complex scores (e.g., APACHE II, Caprini). In the worst-case scenarios, when the LLM attempts to output text to explain itself, it triggers JSON-schema validation errors, causing an endless retry loop that results in `request_limit of 50` exceptions.

2. **How many of the shown failures share this pattern?**
   All 45 of the 45 displayed failures suffer from inaccurate zero-shot calculations. Additionally, the 3 `**exception raised**` failures are explicitly due to the framework crashing on schema retry limits when the LLM attempts to output text.

3. **What is the MINIMAL change that fixes this pattern?**
   We can implement `calculate_medical_value` as a pure Python (`direct`) function that serves as a bridge. Inside it, we call a newly defined `@implement_via('simulate')` tool (`_cot_simulator`) whose return type is `str`. This string schema allows the LLM to freely output step-by-step reasoning. Crucially, unlike Iteration 1 which failed by calling a helper that *did not receive the `patient_note` as an argument*, our `_cot_simulator` explicitly takes both the `patient_note` and the `question`. We then use the already provided `_extract_number` function to pull the final float from the text securely.

4. **Why will this change NOT break currently working cases?**
   It completely preserves the original type signature `calculate_medical_value(patient_note, question) -> float` expected by the evaluator. The `_extract_number` function gracefully falls back to Regex-based extraction (`nums[-1]`) if the LLM forgets the `<answer>` tags, ensuring we always return a valid float and never crash the pipeline. Existing working cases will only benefit from the structured reasoning.
