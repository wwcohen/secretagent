1. **What SPECIFIC failure pattern did you identify?**
   The pipeline currently suffers from two major issues:
   - **Zero-Shot Calculation Errors:** The orchestrator agent is forced to calculate complex medical scores (like APACHE II, Caprini, HEART) zero-shot because the provided step-by-step computational tools (`calculate_clinical_score`, etc.) lack the `@interface` decorator. Without this decorator, the agent framework does not expose them as callable tools in the LLM's schema, making them invisible.
   - **Schema Validation Crashes:** When the agent attempts to explain its math in the final output (compensating for the lack of CoT tools), it violates the strict `float` return schema of `calculate_medical_value`. This triggers JSON-schema validation errors, entering endless retry loops that culminate in `request_limit of 50` exceptions.

2. **How many of the shown failures share this pattern?**
   All 45 failures stem from these intertwined architectural issues. The 42 calculation errors are due to the inability to perform explicit Chain-of-Thought, and the 3 exceptions are due to schema retry crashes.

3. **What is the MINIMAL change that fixes this pattern?**
   - I added the `@interface` decorator to the three induced ptools (`calculate_clinical_score`, etc.). This formally registers them in the Orchestrator's tool schema so they can actually be used.
   - I updated the signatures of these tools from `(focus: str)` to `(patient_note: str, score_name: str)` and updated their internal docstrings so the agent knows exactly how to pass arguments without hacking them together.
   - I updated the docstring of the main entry point `calculate_medical_value` with a `CRITICAL INSTRUCTION` guiding the Orchestrator to *first* call `calculate_clinical_score` for the step-by-step breakdown, and *only then* output the final float. 

4. **Why will this change NOT break currently working cases?**
   Unlike previous iterations that attempted to hijack the execution flow by changing the entry point to a `direct` python implementation (which bypassed the Orchestrator's natural routing and fell back to the 40.8% baseline), this safely preserves `calculate_medical_value` as an `@interface`. It just gives the Orchestrator the tools it previously lacked. Furthermore, by strictly commanding the agent to return a raw float at the end, we prevent schema validation crashes without altering the underlying python pipeline typing.
