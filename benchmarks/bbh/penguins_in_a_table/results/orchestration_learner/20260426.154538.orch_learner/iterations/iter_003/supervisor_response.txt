An analysis of the failures reveals that the current pipeline becomes self-destructive when handling distractor tables (like the giraffes). 

### Identified Failure Pattern:
1. **Destructive Table Replacement (`valid.011`, `valid.034`)**: Due to the previously added "Multiple Tables" extraction rule, the LLM unconditionally issues a "replace the current table with the new giraffe table" action. 
2. **Loss of Primary Data**: When `table_operation` executes this replacement, the primary penguin table is entirely destroyed. If the multiple-choice question subsequently asks about penguins, `answer_question` fails or hallucinates (returning names like "Marian" or "None"). 
3. **Random Guessing on Failure**: Because the invalid/None answer isn't in the options, `choose_response` is forced to guess a random option or returns malformed output, bypassing any possible fallback.

### The Fix:
1. **Change Extraction Strategy to 'Single Table Focus'**: Instead of destructively replacing the table, we explicitly instruct `analyze_input` to ignore unrelated distractor tables. This preserves the primary table so that questions about it succeed normally.
2. **Prevent Random Guessing**: We refine `choose_response` to explicitly return `('None', 'None')` when it encounters an answer that isn't among the options, preventing the LLM from hallucinating a choice.
3. **Robust Zero-Shot Fallback**: We update `penguins_workflow` to intelligently short-circuit exact matches, strictly validate the option letter returned by `choose_response`, and trigger a fallback to the `zeroshot_unstructured_workflow` if the result is invalid or 'None'. The zero-shot workflow processes the entire raw text, naturally handling out-of-distribution queries (such as questions genuinely about the distractor tables).

class PenguinsEvaluator(Evaluator):
    def compare_predictions(self, predicted_output, expected_output) -> dict[str, Any]:
        def normalize(s):
            return str(s).strip().strip('()')
        return dict(correct=float(normalize(predicted_output) == normalize(expected_output)))

# ── hand-coded workflow ───────────────────────────────────────────────────────

def penguins_workflow(input_str: str) -> str:
    """Hand-coded workflow implementing answer_penguin_question.

    To use:
        ptools.answer_penguin_question.method=direct
        ptools.answer_penguin_question.fn=ptools.penguins_workflow
    """
    try:
        table, actions, question, options = analyze_input(input_str)
        for action in actions:
            table = table_operation(table, action)
        answer = answer_question(table, question)
        
        # 1. Try exact match first (case-insensitive, stripped) to avoid LLM guessing
        if answer is not None:
            ans_str = str(answer).strip().lower()
            if ans_str != '' and ans_str != 'none':
                for letter, text in options:
                    if str(text).strip().lower() == ans_str:
                        clean_letter = str(letter).strip("() \n\t").upper()
                        return f'({clean_letter})'
                        
        # 2. Use LLM to cleanly map the answer to an option
        ans_input = str(answer).strip() if answer is not None else "None"
        res = choose_response(ans_input, options)
        
        if res is not None and isinstance(res, (list, tuple)) and len(res) >= 2:
            letter = str(res[0]).strip("() \n\t").upper()
            if letter != 'NONE':
                # Validate the returned letter is actually one of the real options
                valid_letters = {str(opt[0]).strip("() \n\t").upper() for opt in options}
                if letter in valid_letters:
                    return f'({letter})'
                    
        # 3. Fallback to zero-shot if the answer is 'None' or doesn't map to any option
        return zeroshot_unstructured_workflow(input_str)
        
    except Exception:
        # Final fallback for any other exceptions (like unpack errors or extraction failures)
        return zeroshot_unstructured_workflow(input_str)
