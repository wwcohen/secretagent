1. **Specific pattern**: The pipeline is currently running at 100% train and eval accuracy. This is excellent! However, there is an opportunity to optimize performance and cost by avoiding unnecessary LLM calls.
2. **Analysis**: The `consistent_sports` tool currently requires an LLM call for *every* comparison. Many comparisons involve the exact same sports (e.g. `consistent_sports("basketball", "basketball")`) or checking against `"general sports"`.
3. **Minimal change**: Introduce a fast-path python helper `_check_consistency` that first checks for direct string equality (`sport1 == sport2`) or the `"general sports"` wildcard, and immediately returns `True`. If these fast checks fail, it falls back to the `consistent_sports` LLM tool.
4. **No regression risk**: This strictly mirrors the exact rules provided in the `consistent_sports` docstring. It bypasses the LLM only for trivial cases where the result is definitively `True`, maintaining 100% accuracy while improving execution speed and significantly lowering API costs per case.
