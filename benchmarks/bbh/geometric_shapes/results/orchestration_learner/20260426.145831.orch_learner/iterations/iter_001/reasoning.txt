An analysis of the failure cases revealed several major issues in the pipeline:
1. **Dropped Arc Commands:** `normalize_path` relied on a segment extraction function (`_segments_from_commands`) that completely ignored all non-`L` commands (like `A` for arcs). This resulted in shapes like "Sectors" being reduced to single lines, losing critical geometric information.
2. **Missing Side Lengths:** The LLM was trying to distinguish between Kites, Rectangles, and Trapezoids using only angles without explicitly knowing the side lengths. Adding mathematically exact side lengths directly via Python resolves this.
3. **Misaligned Angles:** The old pipeline appended the computed angle *before* the description of the line that arrived at the vertex, confusing the LLM about which angle belonged to which corner. It also failed to compute the final closing angle of polygons.
4. **LLM Cost & Hallucination:** `describe_command` and `compute_angle` made up **60%** of the total cost and occasionally hallucinated math. Moving their logic to pure Python saves cost, speeds up the pipeline, and provides exactly precise lengths and angles.
5. **Ellipse/Circle ambiguity:** The dataset consistently expected `ellipse` when two `A` commands formed a full loop (even if $r_x == r_y$). Added explicit targeted instructions to map this pattern.
