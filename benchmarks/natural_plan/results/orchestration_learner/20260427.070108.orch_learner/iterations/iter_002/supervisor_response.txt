An analysis of the failures reveals a few specific issues that cascade through the pipeline:
1. **Misinterpretation of Directed Flights**: In `parse_trip_constraints`, the LLM consistently parses directed flights (e.g. "from Riga to Tallinn") as bidirectional. This causes the next step to hallucinate paths that aren't actually possible.
2. **Time Window Arithmetic Failures**: The LLM running `find_valid_route` struggles to keep a running sum of dates in its head when it just outputs a JSON array, often selecting routes that violate time windows because it miscalculated the day overlapping offset.
3. **Format Hallucinations**: In `build_trip_plan`, the LLM drops required phrasing (e.g. the first line must explicitly use `"Arriving in {City} and visit..."`), miscalculates day ranges, or hallucinates apologies if it detects a prior step failed.

To fix these issues safely without introducing a risky programmatic DFS (which caused regressions previously):
- I have updated the `parse_trip_constraints` docstring to add strict, explicit rules for handling directed vs bidirectional flights.
- I have updated the `find_valid_route` docstring to force a **Chain-of-Thought (CoT)** reasoning block *before* outputting the JSON array. This helps the LLM calculate overlapping days explicitly and verify time windows.
- I have converted `build_trip_plan` into a deterministic pure Python function. This entirely eliminates hallucinated dates, skipped prefixes, and formatting errors while seamlessly working within the existing orchestrator and cutting execution costs.

"""Task-specific interfaces for NaturalPlan trip planning.

Decomposition derived from LLM reasoning traces:
1. parse_trip_constraints — extract cities, durations, flights, time windows
2. find_valid_route — find city ordering respecting all constraints
3. build_trip_plan — assign day ranges and format itinerary
"""

import json
import re
from secretagent.core import interface


@interface
def parse_trip_constraints(problem_text: str) -> str:
    """Parse a trip-planning problem into structured JSON.

    Args:
        problem_text: The ENTIRE trip-planning problem as a single string
            (total trip days, each city's required stay duration, direct
            flight connections, time-window constraints, etc.). Pass the
            whole problem text as one argument — do NOT split it into
            multiple arguments.

    Returns:
        A JSON string encoding a dict with these keys:
        - total_days: total trip duration (int)
        - cities: dict mapping city name to required stay duration in days
            e.g. {"Helsinki": 5, "Barcelona": 3, "Florence": 5}
        - flights: adjacency list of direct flights
            e.g. {"Helsinki": ["Barcelona"], "Barcelona": ["Florence"]}
            CRITICAL FLIGHT DIRECTION RULES:
            - If the text says "CityA and CityB", the flight is BIDIRECTIONAL. Add CityB to CityA's list AND CityA to CityB's list.
            - If the text says "from CityA to CityB", the flight is STRICTLY DIRECTED. Add CityB to CityA's list, but DO NOT add CityA to CityB's list!
        - time_windows: list of time-window constraints, each with keys
            city, earliest_day, latest_day.

    Example:
        >>> parse_trip_constraints("Visit 3 cities over 13 days: Helsinki 5 days, Barcelona 3 days, Florence 5 days. Flights: Helsinki and Barcelona, from Barcelona to Florence. Meet a friend in Florence between day 9 and 14.")
        '{"total_days": 13, "cities": {"Helsinki": 5, "Barcelona": 3, "Florence": 5}, "flights": {"Helsinki": ["Barcelona"], "Barcelona": ["Helsinki", "Florence"], "Florence": []}, "time_windows": [{"city": "Florence", "earliest_day": 9, "latest_day": 14}]}'
    """


@interface
def find_valid_route(problem_text: str, constraints_json: str) -> str:
    """Find a valid ordering of cities that respects all constraints.

    Args:
        problem_text: The original problem text (same string you passed to
            parse_trip_constraints).
        constraints_json: The JSON STRING output returned by
            parse_trip_constraints. Pass the raw JSON string as-is — do
            NOT parse it into a dict first.

    Returns:
        First, think step-by-step to find the correct route:
        1. List the cities and their durations.
        2. Identify the time windows (e.g. city X must be visited on day A to B).
        3. Trace possible routes. For each route, calculate the exact arrival and departure days:
           - The first city starts on Day 1.
           - End Day = Start Day + Duration - 1.
           - Next City Start Day = Previous City End Day (they overlap on the flight day).
        4. Check if the dates match the time windows EXACTLY.
        5. Verify that valid flights exist between consecutive cities. Check the direction carefully.
        
        Finally, output the final route as a JSON array of strings enclosed in a markdown code block.

    Example:
        >>> find_valid_route("...", '{"total_days": 13, "cities": {"Helsinki": 5, ...}, ...}')
        'Reasoning:\n- Helsinki: 5 days. Day 1 to 5.\n- Barcelona: 3 days. Day 5 to 7.\n- Florence: 5 days. Day 7 to 11.\nValid flights match.\n```json\n["Helsinki", "Barcelona", "Florence"]\n```'
    """


def build_trip_plan(problem_text: str, constraints_json: str, route_json: str) -> str:
    """Assign day ranges to each city and format the day-by-day itinerary.
    
    This replaces the previous LLM call with deterministic pure Python to eliminate
    formatting and arithmetic hallucination errors.
    """
    # 1. Parse constraints safely
    match_c = re.search(r'```(?:json)?\s*(\{.*?\})\s*```', constraints_json, re.DOTALL | re.IGNORECASE)
    if match_c:
        c_str = match_c.group(1)
    else:
        match_c2 = re.search(r'\{.*\}', constraints_json, re.DOTALL)
        c_str = match_c2.group(0) if match_c2 else constraints_json
    try:
        constraints = json.loads(c_str)
    except Exception:
        constraints = {}
        
    # 2. Parse route safely
    match_r = re.search(r'```(?:json)?\s*(\[.*?\])\s*```', route_json, re.DOTALL | re.IGNORECASE)
    if match_r:
        r_str = match_r.group(1)
    else:
        match_r2 = re.search(r'\[.*\]', route_json, re.DOTALL)
        r_str = match_r2.group(0) if match_r2 else route_json
        
    try:
        route = json.loads(r_str)
    except Exception:
        route = []
        
    if not isinstance(route, list):
        route = []
    route = [str(c).strip() for c in route]
    
    total_days = constraints.get("total_days", 0)
    num_cities = len(route)
    
    # Extract from text to ensure correctness even if constraints/route are slightly off
    header_match = re.search(r'(\d+)\s+European cities for\s+(\d+)\s+days', problem_text, re.IGNORECASE)
    if header_match:
        num_cities = int(header_match.group(1))
        total_days = int(header_match.group(2))
        
    if len(route) == 0:
        return f"Here is the trip plan for visiting the {num_cities} European cities for {total_days} days:\n"
        
    lines = []
    lines.append(f"Here is the trip plan for visiting the {num_cities} European cities for {total_days} days:\n")
    
    cities_durations = constraints.get("cities", {})
    
    current_day = 1
    for i, city in enumerate(route):
        # Fallback to text matching if missing
        stay = cities_durations.get(city)
        if stay is None:
            stay_match = re.search(rf'{city} for (\d+) days', problem_text)
            if not stay_match:
                stay_match = re.search(rf'spend (\d+) days in {city}', problem_text)
            if stay_match:
                stay = int(stay_match.group(1))
            else:
                stay = 1
                
        end_day = current_day + stay - 1
        
        if i == 0:
            lines.append(f"**Day {current_day}-{end_day}:** Arriving in {city} and visit {city} for {stay} days.")
        else:
            lines.append(f"**Day {current_day}-{end_day}:** Visit {city} for {stay} days.")
            
        if i < len(route) - 1:
            next_city = route[i+1]
            lines.append(f"**Day {end_day}:** Fly from {city} to {next_city}.")
            
        current_day = end_day
        
    return "\n".join(lines)
