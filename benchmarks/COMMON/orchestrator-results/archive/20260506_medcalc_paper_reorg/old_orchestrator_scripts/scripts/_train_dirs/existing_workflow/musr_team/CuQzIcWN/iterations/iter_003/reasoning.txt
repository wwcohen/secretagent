An analysis of the failure cases and the history of previous iterations reveals a clear path to improvement:

1. **Failure Pattern Identified:** The zero-shot `raw_answer` model consistently fails because it hallucinates holistic constraint satisfaction. It cannot simultaneously juggle hard skill limits (e.g. fear of heat) and soft/hard interpersonal conflicts in a single pass.
2. **Why Iteration 1 and 2 Failed:** Both previous iterations tried to introduce Chain-of-Thought (CoT). However, when a model reasons about choices step-by-step, it naturally quotes the text of those choices. If you pass this entire reasoning block directly to the downstream `extract_index` tool, it acts as a "greedy matcher"—it often just finds the *first* choice mentioned in the text (e.g., Choice 0) and erroneously returns its index, ignoring the actual conclusion.
3. **The Minimal Fix:** We implement the "single-ptool approach bound to simulate with thinking" that the docstring explicitly calls out as the optimal architecture (84% accuracy historically). However, we solve the extraction bug by forcing the LLM to output a highly specific string (`BEST_INDEX: <number>`) and parsing it directly in Python using `re.search`. This completely bypasses the `extract_index` hallucination issue while allowing the model to freely reason and quote choices.
4. **Generalization:** The prompt logically asks the model to isolate conflicts and map constraints, capturing nuances like "toxic" members (who conflict with everyone and must work alone) without hard-coding rules to specific narratives.
