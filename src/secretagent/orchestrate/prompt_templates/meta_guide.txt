## Your Task

Analyze the population's strengths and weaknesses. For each candidate on
the Pareto front, reason about which mutation operator would most likely
improve it.

## Strategy Priorities (most effective to least)

1. **ACCURACY FIRST**: If accuracy < target, focus on accuracy improvement:
   - upgrade: try a STRONGER model (GLM-5.1, MiniMax-M2.7) for ptools with low accuracy
   - evolve: rewrite ptool prompts/implementations using LLM evolution
   - swap_strategy: change method (e.g. simulate -> prompt_llm with custom template)

2. **Python over LLM**: If a ptool's task can be solved reliably with Python code,
   swap_strategy to "direct" with LLM fallback is ALWAYS better than downgrading
   to a cheaper LLM model. Python code is free, fast, and deterministic. Keep the
   LLM as a fallback for edge cases.

3. **Cost reduction only after accuracy is satisfactory**:
   - downgrade: try a cheaper model only when accuracy is already high
   - prune: remove unnecessary ptools

## Considerations
- Which ptools have high error rates? (-> repair, upgrade, evolve)
- Which ptools consume disproportionate cost? (-> swap_strategy to direct first, then downgrade)
- Could a ptool be replaced by Python code + LLM fallback? (-> swap_strategy to direct)
- Could a stronger model improve accuracy? (-> upgrade to GLM-5.1 or MiniMax-M2.7)
- Could rewriting a ptool's prompt/implementation help? (-> evolve)
- Are there patterns in which cases succeed vs fail? (-> route, induce)
- Could combining strengths of two candidates help? (-> crossover)
