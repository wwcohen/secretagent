## Your Task

Analyze the population's strengths and weaknesses. For each candidate on
the Pareto front, reason about which mutation operator would most likely
improve it.

Consider:
- Which ptools have high error rates? (-> repair, upgrade, swap_strategy)
- Which ptools consume disproportionate cost? (-> downgrade, prune)
- Are there patterns in which cases succeed vs fail? (-> route, induce)
- Could combining strengths of two candidates help? (-> crossover)
- Is a ptool hitting its max_tokens ceiling? (-> swap_strategy with higher max_tokens)
- Could an LLM-backed ptool be replaced by deterministic code? (-> swap_strategy to direct)

Return a JSON array of mutation proposals:
```json
[
  {
    "operator": "operator_name",
    "candidate_index": 0,
    "reasoning": "why this mutation on this candidate",
    "params": {"key": "value"}
  }
]
```

Propose 1-3 mutations per iteration. Prioritize high-impact, low-cost
mutations. If the population is already strong and budget is low, propose
no mutations (empty array) to signal convergence.
