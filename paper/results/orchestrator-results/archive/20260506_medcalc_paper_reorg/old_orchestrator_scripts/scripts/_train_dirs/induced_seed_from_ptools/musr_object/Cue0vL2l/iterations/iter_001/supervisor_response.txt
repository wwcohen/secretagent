An analysis of the failure cases reveals a consistent flaw in how the pipeline handles "Theory of Mind" (ToM) tracking, specifically regarding **Inattention Blindness**. 

In almost all the failed cases, the target character is physically present in the room during an object's relocation, but the text explicitly describes them as `"engrossed"`, `"distracted"`, `"immersed"`, or `"busy"` with something else. The base LLM prompt incorrectly assumes that because the character is in the room, they successfully observed the movement. 

Conversely, the LLM sometimes misses direct observations (e.g., when a character explicitly "spotted" a move) or logical deductions (e.g., knowing a coworker took an item for a specific job).

### Strategy
Instead of destructively overwriting the existing `_analyze_character_knowledge_impl` or `_extract_belief_location_impl` (which works well for ~69% of cases), we will introduce a **targeted refinement step** (`_refine_knowledge_analysis`). This step acts as a self-correction filter that strictly enforces the rules of inattention blindness, direct observation, and logical deduction before the final index extraction occurs. This safely addresses the failing edge cases without breaking the syntax or formatting expected by the downstream functions.

<reasoning>
1. **Specific failure pattern:** The base knowledge analysis fails to account for characters who are present but distracted (e.g., "engrossed in a call", "preoccupied with boxes", "pampering"). It falsely updates their belief state as if they watched the object being moved.
2. **Frequency:** ~20 out of the 23 failures exhibit this exact problem of misinterpreting a character's visual attention or missing a logical deduction based on routines/roles.
3. **Minimal change:** Add a new `@interface` function called `_refine_knowledge_analysis` that reviews the preliminary knowledge output and corrects it using strict ToM rules. Insert this step into the workflow immediately before the final extraction.
4. **Safety:** By leaving the original 3 functions and their outputs untouched, we avoid regressions in working cases and preserve the expected string formats. The refinement step simply provides a cleaner, highly-scrutinized context string to the final index extractor.
</reasoning>
