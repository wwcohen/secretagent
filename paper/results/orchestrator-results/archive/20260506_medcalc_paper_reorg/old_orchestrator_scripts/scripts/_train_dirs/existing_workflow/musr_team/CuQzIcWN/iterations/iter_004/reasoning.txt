However, the file already imports `react_solve` from `ptools_common` but never uses it. `react_solve` is the pre-built, correctly-formatted tool for "simulate with thinking" (ReAct). The fix is to surgically replace `raw_answer` with `react_solve` in `answer_question`, while adding defensive type-checking just in case it natively returns an `int`.

It avoids all custom prompt engineering (preventing overfitting/hallucinations), preserves the core Python logic flow, and natively protects against extraction type crashes.
