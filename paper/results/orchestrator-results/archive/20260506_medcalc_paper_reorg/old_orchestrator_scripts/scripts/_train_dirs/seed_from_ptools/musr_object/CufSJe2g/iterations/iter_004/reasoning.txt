1. **Specific failure pattern**: The current `answer_question` function routes directly to `raw_answer`, which is a zero-shot LLM call. This zero-shot inference consistently fails at complex Theory of Mind (ToM) spatial tracking. Specifically, it misses when the target character was unaware of an object moving because they were distracted (e.g., "engrossed", "on a call", "immersed"). It also frequently confuses the target object with other distractor objects that moved, or predicts the old location when the target character *themselves* moved the object.
2. **How many failures share this pattern**: All 33 failure cases trace back to these specific cognitive traps heavily featured in the MUSR dataset generation templates.
3. **Minimal change that fixes this pattern**: Rather than trying to rebuild the entire multi-step workflow (which caused regressions/timeouts in previous iterations) or returning CoT text (which breaks the downstream `extract_index` matcher), we inject a robust set of `CRITICAL THEORY OF MIND RULES` directly into the `narrative` before passing it to `raw_answer`. This explicitly primes the LLM's attention mechanism to map phrases like "engrossed" or "silently" directly to the target person's belief state without altering the expected output format.
4. **Why this won't break working cases**: This does not introduce any new LLM calls (protecting the budget and avoiding timeouts). It doesn't modify the downstream `extract_index` function or its inputs (since `raw_answer` will still output a short string choice). For cases that were already working, it simply reinforces the correct logical deductions. It's a completely safe, purely prompt-based safeguard.
