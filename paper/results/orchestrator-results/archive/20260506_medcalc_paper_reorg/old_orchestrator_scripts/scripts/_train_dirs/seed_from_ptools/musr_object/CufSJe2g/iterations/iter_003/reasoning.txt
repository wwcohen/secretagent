An analysis of the failures reveals a consistent pattern: the pipeline attempts to perform complex Theory of Mind (ToM) spatial tracking using a purely zero-shot LLM call (`raw_answer`). Zero-shot LLMs frequently fail at this because they lose track of distractor objects, miss whether characters were "engrossed" or "in another room" during a move, or even fail to realize when the target character moved the object themselves.

In Iteration 2, a single-step Chain of Thought (CoT) prompt was attempted, but it caused a regression (0.0% accuracy). This is a known architectural issue: `extract_index` is a fragile string-matching function. When fed a long CoT string containing multiple referenced locations (e.g., "It moved from the desk to the shelf, but she thinks it is at the desk"), `extract_index` often matches the wrong string and fails.

### The Fix
To safely solve this without breaking the extraction step, we use a **two-stage LLM approach**:
1. Add a new explicit `@interface` function called `analyze_belief_state`. This function forces the LLM to output a highly structured, 5-step spatial tracking analysis scoped *only* to the target character and target object.
2. Update `answer_question` to prepend this explicit `=== CAREFUL ANALYSIS ===` to the narrative before calling `raw_answer`. 

This guarantees that `raw_answer` receives an airtight logical conclusion, allowing it to output the clean, short string the system requires for `extract_index` to work flawlessly.
